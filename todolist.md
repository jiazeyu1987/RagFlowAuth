  - `P0`：验收阻断项，不完成不能上线。

## 3. 测试总方案（优化后）

## 3.1 单元测试
- 覆盖范围：
  - 权限判定、会话并发、空闲超时。
  - 任务状态机与重试策略。
  - 论文版本与 diff 算法。
  - 查重评分与片段定位。
  - 出网策略引擎（分级、脱敏、拦截、白名单）。
- 目标：
  - 核心服务覆盖率 >= 85%。

## 3.2 集成测试
- 核心链路：
  - 上传 -> 审核 -> 入库 -> 检索。
  - 论文编辑 -> 版本 -> 查重 -> 报告。
  - 外发请求 -> 策略判定 -> 放行/拦截 -> 审计。
  - 采集任务异常 -> 重试 -> 恢复。

## 3.3 E2E（Playwright）
- 角色链路：
  - 管理员全流程。
  - 业务用户全流程。
  - 越权访问阻断。
- 稳定性链路：
  - 大批量任务下其他用户操作可用性。
  - 页面刷新后任务进度恢复。
- UI 链路：
  - 三栏布局关键页面回归。

## 3.4 安全测试
- 脱敏误判测试集（误放行/误拦截）。
- 高敏拦截命中验证。
- 外部模型白名单准入验证。
- 审计完整性与不可抵赖性校验。

## 3.5 性能与稳定性测试
- 混合负载压测（高并发读 + 多任务写）。
- NAS 大文件夹上传压测（验证不阻塞其他请求）。
- 长稳测试（8h/24h）。
- 重启恢复测试（任务状态与进度恢复）。

## 4. 风险、应对与回滚（新增优化）

## 4.1 关键风险
- R1：查重模块引入后计算耗时高，影响主链路。
- R2：出网策略误拦截导致业务中断。
- R3：UI 大改导致学习成本增加、效率下降。
- R4：任务系统迁移期间可能出现状态不一致。

## 4.2 应对策略
- R1：查重任务强制异步，设置并发上限与资源隔离。
- R2：策略先灰度为“审计模式”，观察后切“拦截模式”。
- R3：保留旧布局开关一期，双轨运行 2 周。
- R4：迁移采用双写/校验窗口，发现异常立即切回旧任务通道。

## 4.3 回滚方案
- [ ] 版本级回滚：保留前一稳定版本镜像与数据库快照。
- [ ] 功能开关回滚：查重、出网策略、UI 新布局均可一键关闭。
- [ ] 数据回滚：迁移脚本必须有反向脚本并先演练。
