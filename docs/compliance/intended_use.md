# 预期用途（Intended Use）

版本: v1.0
更新时间: 2026-04-14
范围: RagflowAuth 合规验收基线

## 1. 系统定位

RagflowAuth 用于企业内部受控文档的上传、审批、授权访问、预览、水印展示、电子签名、审计留痕、备份恢复与公司级数据隔离管理。

## 2. 目标用户

- 系统管理员：管理用户、权限组、公司/部门、数据安全配置。
- 上传人员：提交待审文件。
- 审批人员：执行审核、驳回、覆盖审批。
- 查看人员：在授权范围内查看文档、搜索结果与审计证据。
- QA/合规：审查审计日志、验证报告、恢复演练记录。
- 运维：负责发布、备份、恢复与退役执行。

## 3. 预期使用场景

- 文档在受控流程中上传、审批并留痕。
- 文档仅在授权范围内可见。
- 关键审批动作具备电子签名与审计记录。
- 预览/下载行为具备动态水印与可追溯性。
- 不同公司租户的数据物理隔离。
- 备份包可追溯、可恢复、可抽检。

## 4. 非预期用途

- 不作为医疗诊断软件使用。
- 不用于替代正式 DMS/ERP 中的财务、生产执行或设备控制功能。
- 不允许通过共享账号、绕过审批、关闭审计的方式使用。

## 5. 运行前提

- 账号、组织、权限组已正确配置。
- RAGFlow、ONLYOFFICE、邮件/钉钉通道等外部依赖已按运维要求配置。
- 备份目标路径、恢复演练环境和租户数据库路径可访问且受控。

## 6. 合规边界

- 本系统负责实现 `R1-R10` 对应的软件控制与证据归档。
- 培训、职责、供应商评估、变更控制等管理控制通过 `docs/compliance/` 下文件受控管理。

## 7. 维护阶段复核

- 操作系统、数据库、接口、配置和预期用途变更前，必须执行维护影响判定。
- 维护影响判定必须引用 `docs/compliance/traceability_matrix.md` 作为输入，判断是否需要再确认。
- 旧验证结论不得继续沿用；预期用途版本变化后，必须更新验证计划、验证报告和追踪矩阵后才能继续放行。
