在进入详细任务前，先修正一处现状判断，避免后续任务重复:

2. `R8 日志操作记录`
   - 系统已有基础审计、下载日志、删除日志
   - 当前缺的是“合规级审计追踪完整性”，不是“完全没有日志”

### T01 权限现状校正与回归证据

- 修改 `fronted/src/features/review/components/DocumentReviewToolbar.js`
  - 隐藏审批按钮
- 修改 `fronted/src/features/review/components/DocumentReviewTable.js`
  - 隐藏单条审批/驳回/删除按钮

  - 覆盖 upload/review/download/delete 拒绝
- 新增 `fronted/e2e/tests/admin.file-ops-forbidden.spec.js`

### T02 电子签名

目标:
- 把当前“reviewed_by/review_notes”升级为真正的电子签名能力。
- 满足签名绑定、签名含义、签名原因、二次认证、验签留痕。

怎么改:
- 新增“签名前二次认证”接口，要求输入当前密码，换取短时 `sign_token`。
- 审批/驳回/覆盖时必须带 `sign_token`、`signature_meaning`、`signature_reason`。
- 每次签名单独落表，不能只把签名结果覆盖到文档主表。
- 生成记录摘要 `record_hash`，把“签了什么”固定下来。

改后对应文件与条目:
- 新增 `backend/database/schema/electronic_signatures.py`
  - 新建 `electronic_signatures` 表
  - 建议字段:
    - `signature_id`
    - `record_type`
    - `record_id`
    - `action`
    - `meaning`
    - `reason`
    - `signed_by`
    - `signed_by_username`
    - `signed_at_ms`
    - `sign_token_id`
    - `record_hash`
    - `signature_hash`
    - `status`
- 新增 `backend/services/electronic_signature/store.py`
  - 签名写入、查询、验签
- 新增 `backend/services/electronic_signature/service.py`
  - 二次认证、签名摘要生成、签名校验
- 修改 `backend/app/modules/auth/router.py`
  - 新增 `POST /api/auth/signature-challenge`
  - 请求字段建议:
    - `password`
  - 响应字段建议:
    - `sign_token`
    - `expires_at_ms`
- 修改 `backend/models/document.py`
  - `DocumentReviewRequest` 增加:
    - `sign_token`
    - `signature_meaning`
    - `signature_reason`
  - `DocumentResponse` 增加:
    - `signature_id`
    - `signed_at_ms`
- 修改 `backend/app/modules/review/routes/approve.py`
  - 审批前校验 `sign_token`
  - 审批成功后写 `electronic_signatures`
- 修改 `backend/app/modules/review/routes/reject.py`
  - 驳回动作同样走电子签名
- 修改 `backend/app/modules/review/routes/overwrite.py`
  - 覆盖审批同样走电子签名
- 修改 `fronted/src/features/review/api.js`
  - 审批/驳回/覆盖接口 body 增加:
    - `sign_token`
    - `signature_meaning`
    - `signature_reason`
- 修改 `fronted/src/features/review/useDocumentReviewActions.js`
  - 去掉纯 `window.confirm/window.prompt`
  - 改为先弹密码确认，再提交签名
- 新增 `fronted/src/features/review/components/SignatureConfirmModal.js`
  - 输入密码、签名含义、签名原因
- 新增 `backend/tests/test_electronic_signature_unit.py`
- 新增 `backend/tests/test_review_signature_integration.py`
- 新增 `fronted/e2e/tests/review.signature.spec.js`

完成判定:
- 审批、驳回、覆盖都不能绕过二次认证。
- 任意签名都能回查“谁、何时、因为什么、对哪份记录签了什么”。

4. 再做 `T07`、`T08`
   - 把配置变更和备份恢复闭环化

   - 水印和分库改造面更大，放在主链路稳定后推进
