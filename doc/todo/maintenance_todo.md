# 运维体系落地 TODO（稳定优先）

下面是“当前系统”对照 `doc/maintenance/` 运维体系后，仍需补齐的能力清单。

## P0（优先级最高：直接影响稳定与可恢复）

- 运维面板（常驻入口）：状态总览、更新、回滚、日志导出
- 版本化/蓝绿升级落地：A/B 槽位 + 网关切换 + 一键回滚
- 健康检查标准化：部署后自动验证、失败自动回滚
- 诊断包一键导出：配置 + 容器状态 + 关键日志 + 迁移包信息

## P1（强烈建议：降低长期风险）

- 备份闭环：定时策略、保留策略、备份空间预警、恢复演练记录
- 离线镜像保证：迁移包确保包含 `ragflow/images/*.tar`，安装时自动导入
- 监控与告警：磁盘水位（ES/MinIO）、容器重启次数、错误率

## P2（可选：规模化与审计）

- 最小权限与审计：运维操作权限分级、审计日志、内网访问控制
- 更细粒度一致性检查：RAGFlow 组件联通性/索引状态/存储状态巡检
- 运维入口集成第三方工具：Portainer/Grafana 的跳转与单点入口
