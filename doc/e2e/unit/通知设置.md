# 通知设置

## 全真链路状态

- 自动化状态：已接入 `doc/e2e` 全真链路
- 对应自动化：`fronted/e2e/tests/docs.notification-settings.spec.js`
- 运行入口：`python scripts\run_doc_e2e.py --repo-root .`
- 真实数据来源：`scripts/bootstrap_doc_test_env.py` 中 `notifications.history` 与真实渠道配置
- 当前默认规则：`operation_approval_todo` 初始化为“站内信 + 钉钉开启，邮件关闭”，文档断言应与代码默认值保持一致
- 当前已覆盖：规则保存、渠道保存、历史筛选、重试、重发、待处理任务分发
- 当前未覆盖：外部渠道成功送达路径、更多事件类型、更多渠道组合

## 页签目标

用于维护通知规则、发送历史和基础渠道配置，确保系统在业务事件发生时能够按规则分发消息，并可追踪发送结果。

## 适用角色

- 主管理员

## 前置条件

- 当前账号已具备通知设置访问权限。
- 系统中已存在通知规则、发送历史或基础渠道配置数据。
- 若要验证“重试”“重发”等历史操作，应准备至少一条对应状态的发送记录。
- 若外部 SMTP、钉钉等仍为占位凭据，测试应真实记录失败，而不是伪造发送成功。

## 按钮测试项

### 通知规则页签

- 所在区域：页签切换区域。
- 触发前提：当前停留在其他子页签。
- 点击模拟：点击“通知规则”。
- 预期结果：页面切换到通知规则配置视图，显示事件开关、触达对象与规则保存入口。

### 发送历史页签

- 所在区域：页签切换区域。
- 触发前提：当前停留在其他子页签。
- 点击模拟：点击“发送历史”。
- 预期结果：页面切换到发送历史视图，展示筛选区域、历史列表和针对单条记录的补救操作。

### 基础渠道配置页签

- 所在区域：页签切换区域。
- 触发前提：当前停留在其他子页签。
- 点击模拟：点击“基础渠道配置”。
- 预期结果：页面切换到渠道配置视图，展示各通知渠道的基础参数和保存入口。

### 保存通知规则

- 所在区域：通知规则视图。
- 触发前提：已调整通知开关、适用事件或接收对象等规则内容。
- 点击模拟：点击“保存通知规则”。
- 预期结果：当前规则变更被正式保存，刷新页面后仍保持最新配置，后续新产生的业务事件应按新规则分发。

### 分发待处理任务

- 所在区域：通知规则视图。
- 触发前提：系统中存在满足通知规则但尚未分发的待处理任务。
- 点击模拟：点击“分发待处理任务”。
- 预期结果：系统按当前有效规则扫描并分发待处理任务，新增对应的发送结果或待发送记录，已分发任务不应重复停留在待处理状态。

### 查询历史

- 所在区域：发送历史视图的筛选区域。
- 触发前提：已填写发送状态、渠道、时间范围或关键字等筛选条件。
- 点击模拟：点击“查询”。
- 预期结果：历史列表仅展示符合条件的发送记录，结果数量与筛选条件相符，便于管理员追踪特定通知。

### 重置历史筛选

- 所在区域：发送历史视图的筛选区域。
- 触发前提：当前已设置历史筛选条件。
- 点击模拟：点击“重置”。
- 预期结果：所有历史筛选项恢复默认状态，历史列表回到初始展示范围，之前被筛掉的记录重新可见。

### 重试

- 所在区域：发送历史视图的行内操作区。
- 触发前提：任选一条发送失败且允许重试的历史记录。
- 点击模拟：点击“重试”。
- 预期结果：系统基于原消息内容再次发起同一次发送任务，当前记录的状态、重试次数或最新处理结果应同步更新。

### 重发

- 所在区域：发送历史视图的行内操作区。
- 触发前提：任选一条允许人工重新发送的历史记录。
- 点击模拟：点击“重发”。
- 预期结果：系统重新生成一次发送动作，形成新的发送结果或更新原记录状态，目标接收对象应再次收到相同业务消息。

### 查看日志

- 所在区域：发送历史视图的行内操作区。
- 触发前提：任选一条存在发送明细日志的历史记录。
- 点击模拟：点击“查看日志”。
- 预期结果：当前记录的发送过程、失败原因或渠道返回信息被展开显示，便于管理员排查问题。

### 收起日志

- 所在区域：已展开日志的发送历史记录区域。
- 触发前提：当前记录的日志明细已经展开。
- 点击模拟：点击“收起日志”。
- 预期结果：日志明细区域被折叠，列表恢复紧凑展示，其他数据状态不发生变化。

### 保存基础渠道配置

- 所在区域：基础渠道配置视图。
- 触发前提：已修改渠道地址、鉴权信息、开关状态或其他基础参数。
- 点击模拟：点击“保存基础渠道配置”。
- 预期结果：渠道配置被正式保存，刷新页面后仍保留最新值，后续通知发送应按新配置执行。
