# 培训合规

## 全真链路状态

- 自动化状态：已接入 `doc/e2e` 全真链路
- 对应自动化：`fronted/e2e/tests/docs.training-compliance.spec.js`
- 运行入口：`python scripts\run_doc_e2e.py --repo-root .`
- 真实数据来源：`scripts/bootstrap_doc_test_env.py` 中 `training`
- 当前已覆盖：刷新、页签切换、真实用户搜索、培训记录保存、上岗认证保存
- 当前未覆盖：更多培训结果分支、链接预填覆盖、批量历史记录核验

## 页签目标

用于维护培训要求对应的培训记录与上岗认证，帮助管理员跟踪人员培训完成情况和上岗资格。

## 适用角色

- 主管理员

## 前置条件

- 当前账号已具备培训合规访问权限。
- 系统中已存在至少一条培训要求和至少一名可被搜索到的目标用户。
- 若要验证保存动作，应准备可提交的培训结果、认证状态和有效期数据。
- 若要验证链接预填，应准备与现有数据匹配的 `tab`、`user_id`、`requirement_code` 或 `controlled_action` 查询参数。
- 自动化执行时培训要求、培训记录与认证记录必须真实落库，不得伪造成功提示。

## 按钮测试项

### 刷新

- 所在区域：页面顶部工具栏。
- 触发前提：当前已进入培训合规页面。
- 点击模拟：点击“刷新”。
- 预期结果：培训要求、最新培训记录和最新上岗认证列表重新加载为最新状态，页面提示信息按最新请求结果刷新。

### 培训记录

- 所在区域：页内子页签切换区。
- 触发前提：当前停留在“上岗认证”页签。
- 点击模拟：点击“培训记录”。
- 预期结果：页面切换到“录入培训记录”表单和“最新培训记录”列表视图，当前录入上下文保持在培训记录模式。

### 上岗认证

- 所在区域：页内子页签切换区。
- 触发前提：当前停留在“培训记录”页签。
- 点击模拟：点击“上岗认证”。
- 预期结果：页面切换到“录入上岗认证”表单和“最新上岗认证”列表视图，当前录入上下文保持在上岗认证模式。

### 选择目标用户（培训记录）

- 所在区域：培训记录表单中的“目标用户”搜索框与下拉结果区。
- 触发前提：已进入“培训记录”页签，且系统中存在可匹配当前关键词的用户。
- 点击模拟：输入姓名、账号或用户 ID 关键词，并从下拉结果中选择一名用户。
- 预期结果：所选用户被写入当前培训记录表单，后续保存培训记录时会以该用户作为目标用户。

### 保存培训记录

- 所在区域：培训记录表单底部操作区。
- 触发前提：已选择目标用户、培训要求、完成时间，并补充培训结果、有效性评价等必填信息。
- 点击模拟：点击“保存培训记录”。
- 预期结果：系统成功创建一条培训记录，页面出现成功提示，“最新培训记录”列表刷新并展示新记录，表单中可复用的默认值按当前逻辑重置。

### 选择目标用户（上岗认证）

- 所在区域：上岗认证表单中的“目标用户”搜索框与下拉结果区。
- 触发前提：已进入“上岗认证”页签，且系统中存在可匹配当前关键词的用户。
- 点击模拟：输入姓名、账号或用户 ID 关键词，并从下拉结果中选择一名用户。
- 预期结果：所选用户被写入当前上岗认证表单，后续保存上岗认证时会以该用户作为目标用户。

### 保存上岗认证

- 所在区域：上岗认证表单底部操作区。
- 触发前提：已选择目标用户、培训要求、认证状态和有效期，并补充备注等必要信息。
- 点击模拟：点击“保存上岗认证”。
- 预期结果：系统成功创建一条上岗认证记录，页面出现成功提示，“最新上岗认证”列表刷新并展示新记录。
