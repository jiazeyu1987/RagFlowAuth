# 电子签名策略

版本: v1.1  
更新时间: 2026-04-02  
账号唯一性声明: 账号不得共用，不得代签  
仓库外残余项: 线下离岗/转岗签名权限回收记录、账号唯一性培训签收记录

## 1. 基本原则

- 电子签名仅代表签名人本人意图和责任。
- 电子签名必须绑定到当前步骤被授权审批人，不得由仅具备通用 `can_review` 权限但未被当前步骤授权的用户代签。
- 电子签名必须经过二次认证挑战。
- 账号停权、禁用登录、离岗或转岗后，不得继续发起或完成电子签名。

## 2. 适用动作

- 审批通过
- 驳回
- 覆盖审批

## 3. 仓库内控制要求

- 每次签名必须记录:
  - 签名人 `signed_by`
  - 签名用户名 `signed_by_username`
  - 签名时间 `signed_at_ms`
  - 目标动作 `action`
  - 签名含义 `meaning`
  - 签名原因 `reason`
  - 关联业务对象与签名前后快照 `record_payload`
- 签名令牌必须与签名用户一致；不同用户不得消费或复用他人的签名令牌。
- 账号当前不可登录时，系统必须阻断:
  - `issue_challenge`
  - `consume_sign_token`
  - 最终签名落库

## 4. 审批授权绑定

- 授权来源: `approval_workflow_steps`
- 授权方式:
  - `approver_user_id`
  - `approver_role`
  - `approver_group_id`
  - `approver_department_id`
  - `approver_company_id`
- 当前步骤授权校验失败时，接口返回 `approval_actor_not_assigned_to_step`，且不得生成签名记录。

## 5. 审计要求

- 电子签名事件必须进入统一审计日志。
- 审计日志必须关联请求 ID、资源 ID 和 `signature_id`。
- 签名记录必须可验签。

## 6. 仓库外残余项

- 线下离岗/转岗签名权限回收记录
- 账号唯一性/禁共用声明的培训签收记录
- 管理层或 QA 对签名责任矩阵的线下批准记录

上述项目不在仓库内伪造为“已完成”，需由线下受控体系归档。
