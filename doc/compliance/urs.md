# 用户需求规格（URS）

版本: v1.2  
更新时间: 2026-04-03

| URS ID | 对应需求 | 用户需求 |
|---|---|---|
| URS-001 | R1 | 管理员只能执行系统管理和权限分配，不得直接执行文件上传、审批、下载、删除。 |
| URS-002 | R2 | 用户只能查看被授权的文档、知识库和相关结果，不得越权访问。 |
| URS-003 | R3 | 文档预览与受控分发必须带有动态水印，能够追溯操作者和时间。 |
| URS-004 | R4 | 文档审批必须支持多级流程、步骤级审批人或审批范围绑定、待我审批查询、状态推进与审批留痕。 |
| URS-005 | R5 | 审批节点应按当前审批步骤解析对应用户并发送邮件或钉钉通知；通知内容至少包含步骤名、文档名和审批定位字段，并保留重试、补发和结果记录。 |
| URS-006 | R6 | 关键审批动作必须绑定电子签名、签名含义、签名原因和二次认证。 |
| URS-007 | R7 | 系统必须建立受控的 GMP 合规基线，并对关键受控文档、配置变更、追踪矩阵、一致性门禁和周期复核状态提供可追溯证据；仓库外线下签字或真实执行记录不得在系统内被伪造成已完成。 |
| URS-008 | R8 | 关键动作必须形成可查询、可过滤、字段完整的审计日志。 |
| URS-009 | R9 | 不同公司数据必须隔离到不同数据库，避免跨公司读取。 |
| URS-010 | R10 | 备份必须可追溯生成者、生成时间、包 hash、复制校验状态、恢复演练验证状态和验收结果；恢复演练必须由系统执行实际校验并阻断篡改或损坏包的通过。 |
| URS-011 | FDA-02 | 检查取证时必须能够导出审计、电子签名、审批、通知、备份与恢复验证记录的副本；副本需同时提供人可读格式与便携格式，并附带系统生成的完整性摘要。 |
| URS-012 | FDA-03 | 系统文档必须纳入系统可识别、可追踪的受控登记机制，并能按当前发布版本导出最小可用的审查包；仓库外签字批准页、作废回收记录和真实发布批次签核不得在系统内伪造成已完成。 |
| URS-013 | GBZ-01 | 系统必须在维护阶段持续判断操作系统、数据库、接口、受控配置和预期用途变化是否影响已确认状态；命中再确认触发器时必须更新验证证据，预期用途变化时不得继续沿用旧验证结论。 |
| URS-014 | GBZ-02 | 紧急变更必须形成独立受控流程，至少包括申请、指定授权人审批、部署、事后评审与 CAPA 关闭；流程必须先授权后部署，再补齐影响评估和复盘记录。 |
| URS-015 | GBZ-03 | 已批准文档退役后，系统必须在记录保留期内保留可访问的退役记录；仅已授权用户可通过受控入口查询、预览或下载退役文件，管理员仅可通过审计入口导出退役记录包；系统必须记录退役原因、退役人、退役时间、保留截止时间和记录包摘要，且不得以仓库内记录替代线下纸质批准、介质封存和到期处置签字。 |
| URS-016 | GBZ-04 | 系统必须维护已批准供应商/现成软件/第三方组件的受控基线，并记录当前版本、批准版本、供应商审核摘要、已知问题评审和再确认触发；对共享服务、服务器、工作站及租户数据库必须保留环境级 IQ/OQ/PQ 记录，其中租户数据库确认必须绑定 `company_id`，且当组件版本变化时不得继续沿用旧确认状态。 |
| URS-017 | GBZ-05 | 系统必须维护受控培训要求、培训记录、培训有效性评价和操作员认证记录，并对 `document_review`、`restore_drill_execute` 等关键动作执行门禁；当 `curriculum_version` 变化、培训有效性未通过、认证缺失或认证过期时，系统必须阻断关键动作，不得继续沿用旧培训结论。 |
