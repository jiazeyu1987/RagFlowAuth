# 发布与灰度执行手册（Release/Rollout Playbook）

更新时间：2026-03-14  
适用范围：`RagflowAuth` DEV/SIT/UAT/预生产/生产环境发布

## 1. 环境顺序（强制）
1. DEV：开发自测、单元测试、局部 E2E。
2. SIT：集成联调、全量回归、接口兼容验证。
3. UAT：业务验收、权限矩阵验收、关键场景复核。
4. 预生产：与生产同构环境，执行发布演练与监控门禁。
5. 生产：灰度发布与全量放量。

## 2. 灰度策略（10% -> 30% -> 100%）
1. 第一阶段（10% 管理员）：观察任务中心、权限与审计链路。
2. 第二阶段（30% 用户）：扩大至核心业务用户，持续观测错误率和延迟。
3. 第三阶段（100% 全量）：连续观察指标稳定后全量放开。

每阶段至少观察 30 分钟；任一门禁指标触发阈值，立即执行回滚。

## 3. 发布窗口
- 选择业务低峰时段执行发布。
- 生产发布后保留不少于 2 小时观察期。
- 观察期内禁止叠加高风险变更（数据库结构、权限模型、任务调度策略）。

## 4. 监控门禁与自动回滚阈值
建议门禁阈值：
- 错误率 > 2%（5 分钟窗口）
- P95 延迟 > 基线 150%
- 任务积压（`backlog_tasks`）持续 > 50
- 出网拦截率异常突增（较过去 24h 均值 > 2 倍）

触发处理：
1. 立即停止放量。
2. 执行版本回滚（参考 `doc/maintenance/tool_release.md` 回滚流程）。
3. 恢复上一稳定版本并复核核心链路。

## 5. 发布检查单（门禁前置）
发布前必须全部通过：
1. 数据库迁移结果：执行成功、无失败 DDL、具备回滚记录。
2. 核心接口冒烟：鉴权、任务、上传、审核、检索、审计均可用。
3. 权限回归：管理员/业务用户/访客边界通过。
4. 任务恢复验证：重启后任务状态可恢复，异常任务可重试。

建议执行命令：
```powershell
python -m unittest discover -s backend/tests -p "test_*.py"
cd fronted; npm run e2e:all
powershell -File scripts/run_fullstack_tests.ps1
python scripts/run_schema_rollback_drill.py --scope all
```
